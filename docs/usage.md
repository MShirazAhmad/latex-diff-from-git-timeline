# Usage

## In-Editor Selection Flow

### Step 1 — Open the Context Menu

Open your LaTeX project in VS Code and right-click a `.tex` file in **Explorer** or **Source Control** (Changes/History).

Select **Track Changes in LaTeX** from the context menu.

![Simplified VS Code context menu with "Track Changes in LaTeX" highlighted.](./_static/images/latexdiff-full-example-1.png)

*Right-click any `.tex` file in Explorer or Source Control to start the diff workflow without leaving VS Code.*

### Step 2 — Choose the OLD Version

A quick-pick list appears showing commits from the file's git history. Select the commit you want to use as the **older** (baseline) side of the diff.

![Stylized VS Code quick pick for "Select OLD commit" highlighting the choice of a historical commit.](./_static/images/latexdiff-full-example-2.png)

*Anchor the older side of the diff by selecting the commit you want to compare against.*

### Step 3 — Choose the NEW Version

Another quick-pick list lets you pick the **newer** side of the comparison — typically your working directory, or a more recent commit.

![Stylized VS Code quick pick showing the "Select NEW commit" prompt with working directory and recent commits.](./_static/images/latexdiff-full-example-3.png)

*Pick the newer side of the comparison — usually your working directory — to see the latest edits against an earlier commit.*

### Step 4 — Review the PDF Output

The extension writes a `diff_<old>_to_<new>_<file>.tex` file next to your source and offers to open it. Compile it with `pdflatex` (or `xelatex`/`lualatex`) to produce a PDF with tracked changes: **additions in blue**, **deletions in red**.

![Compiled PDF output demonstrating the final tracked-changes document generated by the extension.](./_static/images/latexdiff-full-example-4.png)

*The compiled PDF produced from the generated `latexdiff` file, showing additions in blue and deletions in red just as journals require.*

## Quick Start

1. Open your LaTeX project in VS Code.
2. Right-click a `.tex` file in **Source Control** (Changes/History) or **Explorer**.
3. Select **Track Changes in LaTeX**.
4. Choose OLD and NEW versions (commits or working directory).
5. A `diff_<old>_to_<new>_<file>.tex` is created next to your source file.
6. Compile it: `pdflatex diff_...tex` → open the PDF to see changes highlighted.

## Context Menu Locations

The **Track Changes in LaTeX** command appears in:

| Location | Condition |
|---|---|
| **Source Control → Changes** | File has a `.tex` extension |
| **Source Control → History (Timeline)** | File has a `.tex` extension |
| **Explorer** | File has a `.tex` extension |

## Output File

The generated file is named:

```
diff_<short-old-sha>_to_<short-new-sha>_<original-filename>.tex
```

It is placed in the same directory as the original `.tex` file and is self-contained — it compiles like any standard LaTeX document without additional dependencies (beyond your TeX distribution).

## Customizing Diff Appearance

To change colors (e.g., red → orange for deletions):

1. Open the generated diff `.tex`.
2. Find `\definecolor` or `\textcolor` commands.
3. Adjust the RGB values or color names.
4. Recompile.

If you want the PDF to mirror your journal template, compile with the same main document preamble:

```bash
pdflatex -jobname=tracked-change "\input{diff_...tex}"
```
